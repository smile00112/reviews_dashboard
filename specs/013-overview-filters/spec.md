# Feature Specification: Overview filters — custom date range + brand filter

**Feature Branch**: `013-overview-filters`

**Created**: 2026-07-19

**Status**: Draft

**Input**: User description: "На странице /overview добавить два фильтра: (1) вместо фильтра «Все время» — кнопка «Произвольный диапазон», по клику открывается блок выбора диапазона дат (от/до) для фильтрации; (2) рядом с фильтром «Филиалы» добавить фильтр «Бренды» со списком из таблицы companies; при выборе бренда список в фильтре «Филиалы» сужается до точек этого бренда."

## User Scenarios & Testing *(mandatory)*

### User Story 1 — Произвольный диапазон дат (Priority: P1)

Оператор смотрит «Обзор сети» и хочет метрики не за преднастроенный период, а за конкретный отрезок (например, майские праздники или прошлый квартал). Вместо чипа «Всё время» он видит кнопку «Произвольный диапазон». По клику раскрывается блок с двумя полями дат (от / до) и кнопкой применения. После применения весь дашборд (KPI, распределения, платформы, «Требует внимания», худшие точки, аспекты) считается за выбранный отрезок, а кнопка показывает выбранный диапазон и выглядит активной.

**Why this priority**: Основная просьба; даёт возможность анализа произвольных периодов, которой сейчас нет вообще.

**Independent Test**: Открыть /overview, нажать «Произвольный диапазон», задать от/до, применить — сверить числа с ручной выборкой отзывов за этот отрезок; перезагрузить страницу и убедиться, что диапазон восстановился из URL.

**Acceptance Scenarios**:

1. **Given** оператор на /overview с периодом «30 дней», **When** он нажимает «Произвольный диапазон», **Then** раскрывается блок с полями «от» и «до», предзаполненными текущим активным окном.
2. **Given** блок диапазона открыт, **When** оператор задаёт корректные «от» ≤ «до» и применяет, **Then** все блоки дашборда пересчитываются за интервал [от 00:00; до 23:59:59] включительно, кнопка отражает выбранный диапазон и подсвечена как активный фильтр.
3. **Given** применён произвольный диапазон, **When** оператор кликает любой преднастроенный чип периода (День / Неделя / 30 дней / 90 дней / Год), **Then** произвольный диапазон сбрасывается, кнопка возвращается в неактивное состояние.
4. **Given** применён произвольный диапазон, **When** оператор копирует URL и открывает его в новой вкладке, **Then** дашборд показывает тот же диапазон и те же числа.
5. **Given** блок диапазона открыт, **When** оператор задаёт «от» > «до» или пустое поле, **Then** применение блокируется и показывается понятное сообщение об ошибке, ранее применённый фильтр не меняется.

---

### User Story 2 — Фильтр «Бренды» (Priority: P1)

Сеть состоит из нескольких брендов (компаний), у каждого свои филиалы. Рядом с фильтром «Филиалы» появляется фильтр «Бренды» со списком компаний. Выбор бренда сужает дашборд до его филиалов, а список в фильтре «Филиалы» показывает только точки выбранного бренда.

**Why this priority**: Вторая половина просьбы; без неё оператор вынужден вручную отмечать десятки филиалов, чтобы посмотреть один бренд.

**Independent Test**: Открыть /overview, выбрать бренд — убедиться, что метрики считаются только по его филиалам, а список филиалов сузился до них.

**Acceptance Scenarios**:

1. **Given** оператор на /overview без фильтров, **When** он открывает фильтр «Бренды», **Then** он видит список активных компаний и пункт сброса («Все бренды»).
2. **Given** список брендов открыт, **When** оператор выбирает бренд, **Then** весь дашборд считается только по филиалам этого бренда, а фильтр «Филиалы» содержит только точки этого бренда.
3. **Given** выбран бренд и отмечены его филиалы, **When** оператор переключается на другой бренд, **Then** ранее отмеченные филиалы сбрасываются (они не принадлежат новому бренду) и дашборд показывает весь новый бренд.
4. **Given** выбран бренд, **When** оператор сбрасывает бренд («Все бренды»), **Then** фильтр «Филиалы» снова содержит все точки сети, а дашборд считается по всей сети.
5. **Given** выбран бренд, **When** оператор перезагружает страницу, **Then** выбор бренда восстанавливается из URL.

---

### User Story 3 — Комбинация фильтров (Priority: P2)

Фильтры бренда, филиалов, площадки и периода (в т.ч. произвольного диапазона) работают вместе и не сбрасывают друг друга без нужды.

**Why this priority**: Ценность обоих фильтров падает, если они конфликтуют; но базовая польза достигается и по отдельности.

**Independent Test**: Задать бренд + два филиала + площадку «Яндекс» + произвольный диапазон и проверить, что все четыре ограничения применены одновременно и переживают перезагрузку.

**Acceptance Scenarios**:

1. **Given** заданы бренд, филиалы, площадка и произвольный диапазон, **When** дашборд загружается, **Then** применяются все четыре ограничения одновременно.
2. **Given** заданы бренд и произвольный диапазон, **When** оператор меняет площадку, **Then** бренд, филиалы и диапазон сохраняются.

---

### Edge Cases

- Бренд без филиалов → дашборд показывает пустое состояние (нули/прочерки), а не ошибку.
- Филиалы без бренда (`company_id` не задан) → видны только при сброшенном фильтре брендов.
- «от» = «до» → однодневный диапазон, засчитывается весь этот день.
- Дата «до» в будущем → допускается, фактически ограничивается текущим моментом.
- Слишком ранняя «от» (раньше первых собранных отзывов) → просто пустое начало, не ошибка.
- Диапазон, в котором нет ни одного отзыва → нули и пустые таблицы, без ошибки.
- Список компаний пуст → фильтр «Бренды» показывает пустое состояние и остаётся безвредным.
- Ссылка с несуществующим/некорректным диапазоном или неизвестным брендом в URL → откат на период по умолчанию / сброшенный бренд, без падения страницы.

## Requirements *(mandatory)*

### Функциональные требования

- **FR-001**: На /overview чип «Всё время» ЗАМЕНЯЕТСЯ элементом управления «Произвольный диапазон»; преднастроенные периоды День / Неделя / 30 дней / 90 дней / Год остаются без изменений.
- **FR-002**: Клик по «Произвольный диапазон» раскрывает блок с полями «от» и «до» и явным действием применения; повторный клик / клик вне блока закрывает его без применения изменений.
- **FR-003**: Применённый произвольный диапазон ограничивает интервалом [начало дня «от»; конец дня «до»] включительно ровно те показатели, которые сегодня зависят от выбранного периода (счётчики «новых за период», распределение оценок и тональности за период, показатели скорости ответа за период). Блоки с собственными фиксированными окнами реакции («сегодня», 2ч/24ч, «Требует внимания», тренд аспектов за 14 дней) сохраняют текущее поведение и от выбора периода не зависят — как и сейчас с преднастроенными периодами.
- **FR-004**: Система ДОЛЖНА отклонять применение диапазона при «от» > «до» или незаполненном поле, показывая сообщение и сохраняя предыдущее состояние фильтра.
- **FR-005**: При активном произвольном диапазоне элемент управления ДОЛЖЕН показывать выбранные даты и визуально отличаться как активный фильтр.
- **FR-006**: Выбор любого преднастроенного периода ДОЛЖЕН очищать произвольный диапазон, и наоборот — применение диапазона снимает выделение с преднастроенного чипа.
- **FR-007**: Рядом с фильтром «Филиалы» ДОЛЖЕН появиться фильтр «Бренды» со списком компаний (одиночный выбор) и пунктом сброса «Все бренды».
- **FR-008**: Выбор бренда ДОЛЖЕН ограничивать дашборд филиалами этого бренда.
- **FR-009**: При выбранном бренде фильтр «Филиалы» ДОЛЖЕН перечислять только точки этого бренда; при сбросе бренда — все точки сети.
- **FR-010**: Смена бренда ДОЛЖНА очищать выбор филиалов, чтобы не оставались точки чужого бренда.
- **FR-011**: Выбранный бренд и произвольный диапазон ДОЛЖНЫ отражаться в адресе страницы и восстанавливаться при перезагрузке/переходе по ссылке, как это уже сделано для периода, площадки и филиалов.
- **FR-012**: Некорректные значения бренда/диапазона в адресе ДОЛЖНЫ игнорироваться с откатом к значениям по умолчанию, без ошибки страницы.
- **FR-013**: Фильтры бренда, филиалов, площадки и периода ДОЛЖНЫ применяться совместно (логическое И).
- **FR-014**: Фильтр «Бренды» ДОЛЖЕН показывать активные компании; неактивные компании в списке не показываются.
- **FR-015**: Фича остаётся read-only: фильтры ничего не изменяют в данных и не инициируют сбор отзывов.

### Ключевые сущности

- **Компания (бренд)**: владелец группы филиалов; в фильтре нужны название и признак активности.
- **Организация (филиал)**: точка сети, может принадлежать компании либо не принадлежать никакой; принадлежность определяет содержимое списка филиалов при выбранном бренде.
- **Отзыв**: единица агрегации обзора; дата публикации отзыва определяет попадание в выбранный период/диапазон.

## Success Criteria *(mandatory)*

### Измеримые результаты

- **SC-001**: Оператор задаёт произвольный отрезок дат и видит пересчитанный дашборд не более чем за 3 действия (открыть → задать даты → применить).
- **SC-002**: Числа обзора за произвольный диапазон совпадают с ручным подсчётом отзывов за тот же отрезок в 100% проверок.
- **SC-003**: Оператор ограничивает обзор одним брендом за 2 действия вместо ручного отмечания каждого филиала.
- **SC-004**: Ссылка с любым сочетанием фильтров, открытая заново, воспроизводит тот же экран в 100% случаев.
- **SC-005**: Время отклика обзора с новыми фильтрами не хуже текущего (страница остаётся интерактивной, ощущается мгновенной — порядка текущих ~0.3 с на серверном ответе).
- **SC-006**: Существующие сценарии обзора (преднастроенные периоды, площадки, филиалы) продолжают работать без регрессий.

## Assumptions

- «Вместо фильтра "Все время"» = чип «Всё время» убирается с панели, его место занимает элемент «Произвольный диапазон»; сценарий «за всё время» покрывается заданием достаточно раннего «от».
- Фильтр «Бренды» — одиночный выбор (список компаний невелик), в отличие от множественного выбора филиалов.
- Даты интерпретируются как календарные дни в той же зоне, в которой уже отображаются периоды дашборда; границы включительные.
- Список компаний берётся из уже существующего перечня компаний; новых полей данных не требуется.
- Сравнения «период к периоду» (дельты рейтинга) при произвольном диапазоне используют начало диапазона как точку отсчёта, по аналогии с преднастроенными периодами.
- Прочие страницы (`/reviews`, страницы организаций) не затрагиваются.
- Ограничений по правам нет: обзор доступен тем же пользователям, что и сейчас.
